Study the Access Pattern of Tuples/Pages in a Database System

2014

The database management system (DBMS) manages a massive amount of data, and to process the data, which significantly larger than the main memory, the system required to separate the data into pages unit. There are only a small number of pages load from disk into the memory at a time due to the space limit. The buffer manager is responsible primarily for managing this operation.

In this project, we modified PostgreSql server to track the frequency of each page and tuple that is accessed by the DBMS. Monitoring the rate of each page and tuple accessing helps to manage the buffer more efficiently. For example, if we can place the data items of similar popularities together, the system can reduce the number of access to the disk.
